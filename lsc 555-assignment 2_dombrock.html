Overview on the System

The site holds a description of the Tower with a search box to search within the collection on the top half of the website and if a user scrolls down there are four submenus with images and listing archives of The Tower divided by date using the ranges 1922-1940, 1941-1960, 1961-1980, 1981-2000, ad 2001-2020. The left hand side of the bottom half of the page contains drop down menus to refine results, however, due to the nature of the content in the collection all being of one content type and in a single language, the date menu is the only actual way in which the results can be refined. Somewhat confusingly, the top portion of the website retains a JSTOR search bar, which if used, will search the JSTOR database and not just the collection of The Tower.

Major Strengths and Weaknesses of the System

  • Strengths:
    -Consistency and Standards: For the most part the buttons do what they say they will. Cite/Save/Download options are very easily identifiable and perform as advertised. The search inputs were clearly labeled and it was clear how to search within results, refine results and sort results.
    -Aesthetic and Minimalist Design: Though initially the hosting of the collection into the bigger JSTOR formatting seemed at least a little distracting and potentially confusing, after using the site for a period of time, the minimalist nature of it presented as more of a benefit. I knew to ignore the refine results options except to narrow by date as well as the header menu to search JSTOR content. The actual navigation options within The Tower collection was pretty straightforward and became easy to iterate on searches by using different terms and return to search results quickly. Though there are issues with the actual relevancy of search results, the minimalist design makes the actual navigation and physical act of iterative searching straightforward.
  • Weaknesses:
    -Flexibility and Efficiency of Use: Search tool remains cumbersome to utilize and a lot of extra effort is required by the user to determine if a given result that has a matching term somewhere in the issue is actually of relevance. Additionally, navigation to the relevant area of a given issue is difficult. When a user selects the issue, the main page for the issue is loaded and the user must search within the text manually to find the matching terms. After pulling up multiple articles, it was noted that there is a 'Search matches' submenu on the left side, however, this only notes the page(s) where the search match(es) were noted and the user must still hunt within the page(s), often across multiple articles to find the match and determine its relevancy. This shortfall in search effectiveness is highly inefficient. The image above shows an example of this issue indicating three results within the selected issue where the user is left to hunt the issue text for the actual matches.
    -Help and Documentation: When difficulties were encountered in some initial search queries, one might select the 'Search help' link seeking help searching within The Tower collection, however, the help resources are broad and for JSTOR as a whole entity with no specific help evidently available for The Tower collection specifically. Likewise all information on the 'Contact Us' page are for JSTOR entities and not specific to CUA Collections. These are tradeoffs of hosting the collection on the JSTOR infrastructure, but it seems like there could be some specific information added to the CUA collections section to provide general tips on engaging with the individual collections and a way to contact CUA points of contact for users needing collection specific assistance.

Comparison of Results from Heuristic Review and Mini-Usability Test

The user test also indicated that there was good up front information about the content of the collection and most navigation buttons functioned intuitively. The ability to randomly browse through the archive was quite simple and the user had a positive experience interacting with individual issues of The Tower in the archive. This general ease of navigation and consistency and standards matched generally with the major strengths of the heuristic analysis. Where the heuristic analysis noted that the search tool was cumbersome, the usability test determined it to be so unhelpful that it was abandoned to simple chronological browsing for the desired information. It should be noted that the functionality of the search tools is a complete failure if a user can be turned off so quickly. Perhaps this was underestimated in the heuristic analysis as it was conducted by an individual with more recent experience searching archives and databases whereas for a complete novice, the search tools were even less intuitive or useful. A suggested improvement for the system would be to include some basic instructions or tips to help a user utilize the search tools in an effective way or to provide some helpful resources if the user has difficulties engaging in search.
